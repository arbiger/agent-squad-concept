# SP-Expert 使用範例

以下提供幾個通用範例，展示如何利用 SP-Expert 角色來控管品質與合規性。

---

## 範例 1：品牌與用語規範
**設定內容**：
- 「我們是一家重視夥伴關係的公司。所有產出中嚴禁使用『客戶 (Customer)』，統一更換為『合作夥伴 (Partner)』。」
- 「所有的『Bug』在對外報告中應稱為『待最佳化項 (Optimization Item)』。」

## 範例 2：安全性與開發隱私
**設定內容**：
- 「嚴禁在程式碼中出現任何硬編碼的 API Key 或密碼，即使是測試環境也必須使用環境變數。」
- 「所有用戶數據欄位（如 Email）在日誌紀錄中必須進行去識別化處理。」

## 範例 3：技術架構與效能限制
**設定內容**：
- 「本專案僅支援輕量化部署。所有引入的第三方 Library 必須小於 500KB。」
- 「資料庫查詢必須使用索引，禁止全表掃描。」

## 範例 4：文化與合規策略
**設定內容**：
- 「針對 X 區域的產品，禁止使用特定顏色組合以免觸犯當地習俗。」
- 「所有功能的宣傳文字必須包含免責聲明。」

---

### 使用建議
將以上內容填入 `EXPERT_KNOWLEDGE.md` 中。`Main` 會在啟動時讀取這些規則並交由 `SP-Expert` 進行全程監控。
